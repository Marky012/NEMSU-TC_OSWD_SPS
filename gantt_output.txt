Non-Negotiable Rules
1.  Security first — no feature code is written before Week 2 security layer is complete.
2.  Scope is locked June 13. Explicit out-of-scope: admission system, registrar integration.
3.  Contingency & rollback plan (backup, 4-hr max downtime, step-by-step rollback) must be signed before coding begins.
4.  Question versioning is mandatory — every submission stores a snapshot of questions in submissions.question_snapshot_json.
5.  Email fallback is a development task (Week 4): if SMTP fails after 3 retries, auto-generate downloadable PDF receipt.
6.  Rollback drill is scheduled July 11 — restore from backup and verify data integrity before pilot.
7.  UAT completes July 11 EOD. July 12 is a full buffer day for emergency fixes. Only P0/P1 bugs are fixed.
8.  Post-pilot P2/P3 changes are deferred to Version 2 next semester. Document, don't fix.
